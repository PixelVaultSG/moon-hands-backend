## 4. Architecture Decisions

### Data Storage
- **Multi-tenant with `client_id` separation.** All clinics share tables but rows are tagged.
- NOT separate databases per clinic (overkill for 1-10 clinics).
- Every query filters by `client_id` — no cross-clinic data leakage.

### Smart Router (Hybrid AI)
- **18 intents** handled with $0 hardcoded responses (hours, pricing, services, booking, etc.)
- **Multi-intent support:** "What are your hours and how much is Botox?" → both answers combined
- **Only complex/emotional messages** go to OpenAI GPT-4o mini
- **Saves ~60-70% on API costs**
- Switching to Kimi was evaluated and REJECTED — GPT-4o mini is 4.7x cheaper for our use case

### AI vs Hardcoded (confirmed decision)
- We DO use OpenAI for: complex comparisons, emotional support, ambiguous questions
- We DO NOT use OpenAI for: hours, pricing, service lists, greetings, simple booking — handled by smart router

## 7. My Mistakes (Never Repeat These)

| Mistake | Impact | Fix Applied |
|---------|--------|-------------|
| Said 360dialog was €49/clinic | User would have been shocked by €500 platform fee | Corrected to Phase 1 Direct, we manage accounts |
| Snapshot para 10 said client enters credit card | Wrong workflow | Corrected to WE create account, OUR card |
| Migration SQL had `USING (true)` policies | Any user could read patient waitlist data | `SECURITY_REMEDIATION.sql` dropped them |
| Migration SQL had `ON DELETE CASCADE` on waitlist | Deleting clinic = permanent patient data loss | Changed to `SET NULL` |
| `SUPABASE_SERVICE_KEY` vs `SUPABASE_SERVICE_ROLE_KEY` mismatch | Server would crash on Render | Standardized to `SUPABASE_SERVICE_ROLE_KEY` |
| Webhook had random fallbacks for API_KEY | Silent security bypass | Removed fallbacks, server exits hard |
| `.env` file contained real credentials in code dir | Leak risk | File cleaned, all credentials rotated |
| Greeting regex blocked price queries in simulator | Bot couldn't answer "Hi, how much is Botox?" | Only greet if message ≤3 words |
| Simulator had zero context memory | "Today" after Botox mention → generic fallback | Stored last 3 conversation turns |
| `CREATE TRIGGER IF NOT EXISTS` syntax error | Migration SQL failed | Used `DROP TRIGGER IF EXISTS` + `CREATE TRIGGER` |
| Verification query used wrong table (`pg_tables` not `pg_class`) | RLS check returned error | Fixed to `pg_class` with `relrowsecurity` |
